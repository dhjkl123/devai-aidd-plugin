## What Went Well

1. **Bootstrap discipline.** `src/index.js` stayed as a thin wiring file. Hooks delegate to services. No global mutable state. The structure survived Epic 2's 5-fold expansion of the approval/recovery surface.
2. **Best-effort audit baked in early.** Per-sink try/catch in `src/audit/logger.js` (Story 1.1 era) meant Story 2.5's recovery emissions never had to worry about audit sinks aborting workflow continuation.
3. **Schema-first config.** Story 1.3's Ajv contract caught a forward-compat regression risk before Epic 2 could compound it.
4. **Branch strategy as proposal, not action.** Story 1.4's "proposal-first, no Git mutation in planning" rule prevented Epic 2 from stumbling into the same trap.
5. **Sprint-change-proposal pattern.** Story 1.1's deferral of AC3 to Story 4.2 was documented, traceable, and consensus-driven — not a silent scope cut.

## Challenges (Without Blame)

1. **Story 1.3 review depth surprised the team.** P0 findings on validation recovery required an algorithm redesign mid-review. Lesson: when a story owns a *contract* (config schema in this case), allocate review effort proportional to its blast radius, not its visible code size.
2. **Documentation drift across review rounds.** Architecture.md and sprint-status.yaml weren't always updated alongside code changes. Caught and fixed in Epic 2 round-2 / round-3, but the habit started in Epic 1.
3. **AC3 ambiguity (Story 1.1).** "Bootstrap also generates compatibility files" was technically inside scope but conflicted with the read-only bootstrap principle. The conflict surfaced only in code review, which is late.

## Lessons Learned

- **Auto-fix LOWs in the same review session.** Splitting LOW remediation into a separate session adds ceremony without adding safety. Same-session auto-fix kept the loop tight while still routing every finding through `Review Follow-ups (AI)`.
- **Round-2 / round-3 reviews matter.** Each subsequent round caught issues the previous one missed (documentation drift, defense-in-depth gaps). Don't stop at round 1.
- **`{ outcome, reason, message, details }` envelope.** Standardized in 1.5 readiness, became the lingua franca for Epic 2.
- **`sprint-change-proposal-YYYY-MM-DD.md` is a real artifact, not a temporary note.** Use it whenever scope shifts mid-epic.
